name: sentry-fix
description: 指定したSentryのIssueを現在のブランチで修正してpushする。IssueIDが未指定の場合は対話して確認する。
argument-hint: "[SentryのIssueID]"
---

## 前提確認
作業開始前に以下を確認する：
- 現在のブランチ名を `git branch --show-current` で取得して表示する
- main / master / develop など保護ブランチの場合は「保護ブランチで実行しようとしています。続行しますか？」と警告して確認を取る

## 手順

### 1. IssueIDの確認
引数（$ARGUMENTS）が空の場合：
- Sentry MCP でプロジェクト一覧を取得して表示する
- 「修正したいIssueのIDを教えてください」と聞く
- ユーザーの回答を待つ

引数が指定されている場合はそのIDをそのまま使う。

### 2. エラーの調査と方針確認
- 指定IssueIDの詳細・スタックトレースを取得する
- 原因と修正方針を提示する
- 「この方針で修正しますか？」と確認を取る
- NOなら終了する

### 3. 修正の実装
- 現在のブランチのまま修正する（ブランチは切らない）
- テストがあれば実行して通ることを確認する
- コミットメッセージ: `fix: {エラーの概要} (Sentry: $ARGUMENTS)`

### 4. Push
- `git push origin HEAD` で現在のブランチをpushする
- pushが失敗した場合は原因を報告して終了する

### 5. 完了報告
- 修正したファイル・コミットハッシュ・pushしたブランチ名を報告する
